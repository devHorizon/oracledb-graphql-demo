What GraphQL is

  • An API Specification (like REST)
  • A Query Language (like SQL)
  • A tool to solve problems
  • New (released in 2015) [5]

What GraphQL isn't

  • A piece of software
  • Blockchain
  • AI

The GraphQL Schema for our app

If you're used to RDBMS's this schema will look unusual, albeit the schema makes sense! The corresponding definition is given as defined with SQL

GraphQL's succint syntax allows for a level of expressiveness that language designers strive for and programmers love (which might be one reason it's catching on so quickly). The format is purposely very similar to JSON which is the lingua franca of web APIs. 

type User {
    id: ID! @unique
    email: String!
    idEncoded: String!
    fullName: String!
    fullNameAnotherWay: String!
    posts: [Post!]!
    comments: [Comment!]!
    following: [User!]!
    favNums: [Int!]!
    numLegs: Int!
    luckyNumber: Int!
    writtenMaterial: [AuthoredInterface!]!
}

type Post {
    id: ID! @unique
    body: String!
    author: User
    authorId: Int!
    comments: [Comment!]!
    archived: Boolean!
    createdAt: String!
}

type Comment {
    id: ID! @unique
    body: String! 
    likers: [User!]!
    post: Post!
    author: User!
    authorId: Int!
    archived: Boolean!
    postId: Int!
    createdAt: String!
}

type AuthoredInterface {
    id: ID! @unique
    body: String!
    authorId: Int!
}

Queries

Just like a SQL query you can do selections.

Let's SELECT full name (generating it two different ways) and the body of any written material a user has written
FROM the users table WHERE the users id is 1

{
 user(id: 1) {
  fullName
  fullNameAnotherWay
  writtenMaterial{
    body
  }
 }
}

Variables and Aliases and Fragments, Oh My!

Let's combine them in a pretty complex little query... we want all the information as above for two users, but Variables are self explanatory and aliases are just as the name suggests.the JSON key for each user should be different (that's the alias part). Take a quick look at the below photo and notice the "QUERY VARIABLES" tray. This may be collapsed on your GraphsiQL browser.

And the Query as text for your COPY+PASTE'ing convenience 

fragment foobar on User {
  fullName
  fullNameAnotherWay
  writtenMaterial{
    body
  }
}
query MyQuery($foo: Int!, $bar: Int!) {
  numberOne: user(id: $foo) {
    ...foobar
  }
  numberTwo: user(id: $bar) {
    ...foobar
  }
}

Query Variables definition 

{
  "foo": 1,
  "bar": 2
}

Wow! the Result. Aliases and Variables are indispensable in SQL. I could get use to this "Fragments" too!

Introspection

Introspection to get Schema definitions

{
  __schema {
    types {
      name
    }
  }
}

Which is super useful for developers unframiliar with your API

{
  "data": {
    "__schema": {
      "types": [
        {
          "name": "Query"
        },
        ...
        {
          "name": "__DirectiveLocation"
        }
      ]
    }
  }
}
